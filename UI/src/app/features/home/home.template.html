Overview

As a front-end developer for a financial services company, you will be expected to demonstrate proficiency in AngularJS (version 1.x). The following assessment is designed to test your understanding of AngularJS controllers, services, and common AngularJS directives.

Your task is to create a simple banking app. The app should consist of an account list view, an account detail view, and a popup containing a form. We have provided a service that can be used to retrieve account data. The app should satisfy each point outlined in the acceptance criteria, but we also strongly encourage you to follow the technical criteria to showcase good coding practices, intuitive user experience, and pleasant visual aesthetics.

Acceptance Criteria:

As an application user, I should be able to:

  • See a list of bank accounts that displays the account name, owner name, created date, checking amount, and savings amount. The column values should be formatted appropriately for date and currency values. Ex: $1,200.52, 02-01-2025.
  • Filter the list based on a string query. The filter should return a subset of the full list. Matching should be based on whether a column value contains the filter term. Filter results should ensure there are no duplicates. Ex: If the filter term is "abc", the list should be filtered to display accounts containing "abc" in one of their columns.
  • Sort the list by clicking on any of the column headers.
  • Remove entries from the list via a dedicated delete button.
  • Add new entries to the list via a dedicated button. Clicking the button should open a popup containing a simple form that takes in account name, owner name, checking amount, and savings amount. The created date will be automatically generated at the time the entry itself is created.
  • I can click on any entry from the list view and be taken to a new page containing the detailed view of the account.
    • The detailed view's URL should reference the accountId to retrieve the corresponding account details object included in the detailedAccountList and display the information on the page.
    • Data to be displayed: Owner name, owner age, owner's FICO score, account's checking and savings amounts, and a list of account activity.

Technical Criteria:

We would like to see:

  • Good development practices that provide clear and concise frontend code.
  • A responsive design that can adapt to multiple screen sizes (960px <-> 1920px).
  • Styling achieved using TailwindCSS.
  • Thoughts and consideration to the overall user experience & visual aesthetics of the app.

Notes:

  • The project is written in TypeScript to provide you with good intellisense. However, you are not required to utilize TypeScript (although you may do so if you prefer).
